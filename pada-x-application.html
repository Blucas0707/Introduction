個人簡介

目前任職於半導體業，是一名製程工程師。
在學生時期就接觸過程式，最初是從Ｃ語言入門，到研究生涯的Matlab，再到自學的Python & SQL。因此對程式產生了不小的興趣，而這份火花一直在心底，不曾熄滅。
在工作上使用VBA ,SQL & Python來撈取data，來做後續分析，而在工作之餘也寫過爬蟲，來學習如何爬取資料，也幫朋友架過Line bot，來作為社團間資訊分享的方式。
而近期發現各產業都對數位轉型與網路行銷的人才有著滿大的缺口，而這也暗合我對網站工程師的不斷精進的目標，因此決定參加此計劃，來精進自身能力，尋求下一個職業發展。

成為網站工程師的動機

起初只覺得爬蟲很酷，但隨著深入了解，才發現需要大量前後端的網站架設知識。
同時，因為家中從事傳統產業，近年來也在尋求數位轉型，對於網站的建置、數位行銷甚至到通路金流串接等，都需要相關網站知識。
曾經試著用Wordpress + Siteground 來製作網站，但發現模板方便的同時，也帶來極大的侷限，而對於前後端串接的一知半解，更是可能會有潛在的風險與維護困難。
而我自己曾經試過串接line Message API 及使用Heroku PaaS的經驗，同時也因為工作所需，對於SQL也有初步的認識，所以原先是想走後端工程師。
然而，現在的我，更想將前端的知識ㄧ併整合，未來目標成為一位全端工程師！

對於網站工程師的想像與職涯規劃

我想像中的網站工程師，會是一天的工作時間裡，花1/3時間在與其他團隊做溝通協調，1/3的時間作為專案產品開發，而最後1/3是在團隊技術分享與自我精進。
由於一個產品都需要團隊合作去完成，因此溝通協調、專案分工變得極為重要，因此除了技術過硬以外，溝通能力也必不可少。
同時，我也認為工程師的工作就是要去解決問題，並在問題解決後，會有滿滿的成就感，看到產品誕生，就像看到孩子成長般開心！
對於未來的職涯，我將其規劃為短期2年內與中期(3~5年）及長期（5年以上），共三個目標。
  1. 短期（2年內） ：著重在個人與產業的接軌，了解網路產業的技術發展，與未來趨勢，同時藉由技術論壇、團隊合作等，做跨領域學習
  2. 中期（3~5年）：著重在個人品牌建立與產業的宏觀視野，在建立個人技術的品牌外，希望能夠對產業有足夠的認識，並結合技術去做創新，為社會帶來一些便利
  3. 長期（5年以上）：著重在團隊建立與技術傳承，將自身經驗傳承給未來的工程師，同時培養自身領導能力，成為掌握大方向的管理者

學習計畫

若是有幸參與到此計劃，我會做以下安排：
  • 課程內容分級：針對課程內容，用熟悉程度來做分級，分為熟悉、有點陌生、完全沒碰過，來做學習時間的安排
  • 問題紀錄：將學習過程中的問題與解答紀錄，前期的紀錄，可以讓經驗得以累積
  • 目標追蹤：用專案管理Trello來規劃並追蹤每日須完成目標
