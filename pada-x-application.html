●成為網站工程師的動機

起初只是不想每天被埋在大量的重複性routine工作，花費大量的工作時間在生產上的交接程序（每天至少1~2小時）因此為了增進工作效率，開始自學起程式，開始使用VBA配合SQL撈取data，用python畫成圖表，並結合RPA(Robotic Proces Automation)自動化程序，並分享給課內同仁，提升整個Team的工作績效，讓我充滿成就感，讓我想探索更多不同的領域。

同時，原先家中從事傳統產業，近年來也在積極尋求數位轉型，對於網站的建置、數位行銷甚至到通路金流串接等，都需要相關網站知識，也誘發了我對於網站工程師嚮往的契機。

曾經試著用Wordpress + Siteground 來製作網站，但發現模板方便的同時，也帶來極大的侷限，而對於前後端串接的一知半解，更是可能會有潛在的風險與維護困難。

此外也有串接line Message API 及使用Heroku PaaS的經驗，同時也因為工作所需，對於SQL也有初步的認識，所以原先是想走後端工程師，現在的我，更想將前端的知識ㄧ併整合，未來目標成為一位全端工程師，並將所學知識，幫助更多需要數位轉型的傳統商家，讓更多好的產品與技術被看見！讓傳統工藝能夠得以流傳！

●對於網站工程師的想像與職涯規劃

我想像中的網站工程師，一天的工作時間裡:

  • 1/3的時間在與其他團隊做溝通協調
  • 1/3的時間作為專案產品開發
  • 1/3的時間是在團隊技術分享與自我精進

由於一個產品都需要團隊合作去完成，因此溝通協調、專案分工變得極為重要，因此除了自身專業以外，溝通能力也必不可少。

同時，我也認為工程師的工作就是要去解決問題，並在問題解決後，會有滿滿的成就感，看到產品誕生，就像看到孩子成長般開心！

對於未來的職涯，我將其規劃為短期2年內與中期(3~5年）及長期（5年以上），共三個目標。
  1. 短期（2年內） ：著重在個人與產業的接軌，了解網路產業的技術發展，與未來趨勢，同時藉由技術論壇、團隊合作等，做跨領域學習
  2. 中期（3~5年）：著重在個人品牌建立與產業的宏觀視野，在建立個人技術的品牌外，希望能夠對產業有足夠的認識，並結合技術去做創新，為社會帶來一些便利
  3. 長期（5年以上）：著重在團隊建立與技術傳承，將自身經驗傳承給未來的工程師，同時培養自身領導能力，成為掌握大方向的管理者

●學習計畫與時間分配

目前仍在職，已經規劃在二月底離職，因此若是有幸參與到此訓練營，會有以下規劃：

學習方向

  • 課程內容分級：針對課程內容，用熟悉程度來做分級，分為熟悉、有點陌生、完全沒碰過，來做學習時間的安排
  • 問題紀錄：將學習過程中的問題與解答紀錄，前期的紀錄，可以讓經驗得以累積
  • 目標追蹤：用專案管理Trello來規劃並追蹤每日須完成目標

時間分配

每天至少固定八小時學習，與不限時的課後side project

  • 0.5小時：規劃本日學習進度與需達成的目標
  • 3.5小時：專心學習基礎知識並記錄問題
  • 1.0小時：休息，讓腦袋活絡，迴響與歸納先前的問題
  • 3.0小時：將今日所學實際應用練習，從中尋找不熟悉與不會的地方
  • 1.0小時：今日總結，與明日課程安排
  • 課後：Happy Side Project Time

●預期的困難與解決計畫

將可能遇到的困難，分為三類:

  1. 不容易釐清問題所在，花費大量時間與精力，卻沒有找到方向
  2. 容易被外界的事物分心，無法集中精力
  3. 無法將所學知識，實際應用，導致空泛

解決計畫:

  1. 回到基礎，了解背後原理及應用，並做註記，隨著經驗值累積，會越來越得心應手
  2. 將時間做15分鐘為段落的分配，讓自己能夠時時檢查自己的狀況與進度
  3. 除了此計畫會有專案外，也可以將所學做成side project，增加自己的能力外，也能從中獲得成就感！
